# Design: `/aside` Skill

Capture drive-by ideas without interrupting the current conversation. Optionally dispatches a background subagent that researches and expands the idea so you come back to something richer than what you typed.

## Motivation

During focused work, ideas strike that are unrelated to the current task. Today the options are: lose the thought, break flow to braindump it, or save a vague auto-memory that doesn't really fit any memory type. `/aside` fills the gap — a fire-and-forget capture that parks the idea as a file you can return to later, optionally enriched with research.

This is distinct from `/braindump`, which is an inline conversation that captures session reasoning. `/aside` is non-interactive by design.

## Skill Interface

```
/aside we should look into caching refresh tokens client-side
/aside --quick just a note about the deploy pipeline
```

Everything after `/aside` (or `/aside --quick`) is the user's thought. No name parameter — the agent (or skill, in quick mode) generates a filename slug from the content.

## Two Modes

### Default: Researched

Dispatches the `aside-researcher` subagent in the background. The main conversation continues uninterrupted.

The subagent receives:
- The user's verbatim text
- Locked shared context (auto-injected by the SubagentStart hook)
- Target directory and file naming instructions
- The output document template

The subagent:
1. Preserves the user's original text verbatim — never paraphrases
2. Searches the workspace shared context for related prior art
3. Searches project repo(s) in `repos/` for relevant code, patterns, and docs
4. Searches the web if the topic references external technologies or patterns (agent's judgment)
5. Writes exactly one `local-only-{slug}.md` file to `shared-context/{user}/`
6. Reports back: file path, one-line summary

The subagent does NOT:
- Modify any existing files
- Create more than one file
- Paraphrase or rewrite the user's original thought
- Dispatch nested subagents

### Quick: Note Only (`--quick`)

No subagent. The skill itself:
1. Generates a slug from the content
2. Writes the file directly to `shared-context/{user}/`
3. Reports the file path

Fast, minimal, zero research. For when you just want to park a thought.

## Agent Definition: `aside-researcher.md`

Lives in `.claude/agents/aside-researcher.md`.

| Field | Value |
|-------|-------|
| Model | Sonnet |
| Allowed tools | Read, Glob, Grep, WebSearch, WebFetch, Write |
| Disallowed tools | Edit, Bash, Agent |

The agent definition contains the research contract and output template. The skill constructs a prompt with the user's text and dispatches to this agent via the Agent tool with `run_in_background: true`.

## Document Templates

### Research Mode

```yaml
---
state: ephemeral
lifecycle: active
type: braindump
variant: aside
author: {user}
updated: {YYYY-MM-DD}
---

## User's Original Thought
{Verbatim text from the user — never paraphrased, never modified}

## Agent Research
{Findings from the workspace, project repos, and web.
References specific file paths and prior art.
Structured as prose or sub-headings as the content demands.}

## Synthesis
{How the user's thought connects to what was found.
Proposed next steps, design considerations.
Clearly framed as agent analysis, not user intent.}

## Further Investigation
{Threads worth pulling on. What the agent couldn't answer.
Topics that would benefit from deeper exploration or user input.}
```

### Quick Mode

```yaml
---
state: ephemeral
lifecycle: active
type: braindump
variant: aside
author: {user}
updated: {YYYY-MM-DD}
---

## User's Original Thought
{Verbatim text from the user}

## Further Investigation
{Quick inference of threads worth exploring later.
Generated by the skill, not a subagent.}
```

## File Naming and Location

- **Location:** `shared-context/{user}/local-only-{slug}.md`
- **Slug:** Agent/skill-generated from the content. Kebab-case, descriptive. E.g., `local-only-refresh-token-caching.md`. If a file with that name already exists, append a numeric suffix (`-2`, `-3`).
- **Prefix:** Always `local-only-` — gitignored by default
- **Session-agnostic:** Asides never go to inflight, regardless of whether a work session is active

## Lifecycle

Asides are local-only files. They don't enter version control or shared context until deliberately promoted.

- **`/promote`** — already scans for `local-only-*` files. Aside files are automatically discoverable. Promotion moves the file out of `local-only-*` and into shared context.
- **`/maintenance`** — can flag stale asides that haven't been promoted or acted on.
- **`variant: aside`** frontmatter distinguishes them from other local-only files for tooling and filtering.

Asides never:
- Appear in inflight trackers
- Get swept into `/complete-work` synthesis
- Auto-commit
- Modify existing files

## Interaction with Existing Skills

| Skill | Interaction |
|-------|-------------|
| `/braindump` | No change. Braindump remains inline and conversational. `/aside` is the non-interactive counterpart. The `side` variant in braindump can be deprecated or left as an alias that redirects to `/aside`. |
| `/handoff` | None. Handoffs track workstreams; asides park ideas. |
| `/promote` | Discovers asides via `local-only-*` scan. No changes needed. |
| `/maintenance` | Can surface stale asides. May need a minor addition to check for old `local-only-*` files with `variant: aside`. |
| `/start-work` | No interaction. Asides are session-agnostic. |

## What This Doesn't Cover

- **Conversation context injection:** The subagent does not receive conversation history. The user provides context inline as part of their thought. This is a deliberate simplification — no magic summarization, no prompt bloat.
- **Deep research mode:** No `--deep` flag. The default research mode uses the subagent's judgment on scope. If findings are thin, the Further Investigation section naturally surfaces that gap.
- **Editing existing asides:** No update flow. If the user returns to an idea, they start a new braindump or work session. Asides are seeds, not living documents.
