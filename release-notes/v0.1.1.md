# v0.1.1 Release Notes

**Date:** 2026-04-03

## Features

### Migration tool and workspace initialization

Rewrote the migration module to cleanly separate structure installation from content population. Running `--migrate` installs template structure only — skills, hooks, rules, agents, recipes, settings, workspace.json, CLAUDE.md, and git init — with `workspace.initialized: false` and a `templateVersion` stamp from package.json. Running `--migrate --update` upgrades existing template workspaces by comparing each file against the template, adding missing files, prompting before overwriting modified files, and preserving `local-only-*` files.

Added the `/workspace-init` skill as a guided brainstorming session that populates content after migration. It works on a dedicated branch with granular commits, identifies documentation sources before making a plan, cleans the root directory by moving non-template items to `.claude-scratchpad/unmigrated/`, and runs mandatory self-contradiction verification across all created files. Added a Notion migration recipe (`.claude/recipes/migrate-from-notion.md`) as optional guidance.

### Extended hook system

Expanded the hook system from two to four hooks. PreCompact fires before context compression and prompts the user to capture decisions via `/braindump` or `/handoff` before conversation details are lost. PostCompact fires after compression with a reminder. Updated SessionStart to fire on both fresh starts and resumed sessions (removed startup-only matcher). Added SubagentStart hook to inject `shared-context/locked/` into every subagent automatically. Added PreToolUse repo-write detection hook that warns when writing to project repos on default branches without an active work session. All hook paths now use `$CLAUDE_PROJECT_DIR` instead of relative paths — relative paths fail silently because hooks don't run from the project directory.

## Fixes

### /complete-work inflight cleanup

The `/complete-work` skill previously marked branch handoffs as `lifecycle: resolved` but left them in `shared-context/{user}/inflight/` indefinitely. Step 9 now has explicit disposition rules: branch handoffs are either moved to ongoing (if they contain useful content beyond release notes) or deleted, session braindumps are moved to ongoing or deleted based on scope, and inflight/ must be empty when /complete-work finishes.

## Maintenance

### Spec v2 alignment

Comprehensive update aligning the template with the v2 spec, incorporating 20 design changes from dogfooding. Rewrote all six core workflow skills for dual-repo operations: `/start-work` now branches both the project repo (with worktree) and the workspace repo for traceability, with retroactive formalization for mid-session starts and stale worktree detection. `/complete-work` processes both repos with three-source reading (spec + handoffs + commits). `/handoff` and `/braindump` default to user-scoped context. Added `/sync-work` for pushing both repos without ceremony, `/audit` for on-demand integrity checks, and `/migrate` as both a skill and scaffolder command.

Added four optional rules: scope-guard (detect scope creep), token-economics (cost-conscious model selection), agent-rules (subagent conventions), and context-discipline (capture prompts at breakpoints — later merged into mandatory memory-guidance). Updated the CLAUDE.md template, workspace.json template (greeting, releaseMode fields), and scaffolder prompts.

### Skill improvements

Redesigned `/promote` with a coded table UX (AM1, LOC1, LOR1) showing all candidates from all sources with assessments and recommendations in a single view, plus drop support for removing stale entries. Fixed `/start-work` branch creation to use `git branch` instead of `git checkout -b`, preventing worktree creation failures. Added work session marker files (`.claude-scratchpad/.work-session-{slug}`) for lightweight state tracking across the work session lifecycle.

Renamed `/sync` to `/sync-work` for consistency with the work session skill naming pattern (`/start-work`, `/pause-work`, `/complete-work`, `/sync-work`).

### Workflow maintenance

Renamed `/audit` to `/maintenance` with three modes: `audit` (read-only integrity checks), `cleanup` (stale context, suggested merges, reconciliation), and `full` (both). Moved capture-time cross-checking from `/handoff` and `/braindump` into `/maintenance cleanup` — unreliable as an ambient prompt obligation. Promoted `memory-guidance` from optional to mandatory, merging in content from the deleted `context-discipline` rule. Updated the scaffolder's optional rules list.

## Known Limitations

- **`@shared-context/locked/` directory import** — unverified whether Claude Code supports directory-level `@` imports or requires individual file listing
- **Repo-write hook is warn-only** — does not block writes to repos on default branches, only warns
- **No memory-related hooks** — unknown whether Claude Code exposes hooks for memory writes, limiting auto-promote
- **Template updates for forked teams** — no established pattern for cherry-pick, subtree, or upgrade path
- **Agent rules placement** — undecided whether agent-specific rules belong in a separate directory or use frontmatter scoping
- **Orphaned work session markers** — no automatic cleanup when a session exits without `/complete-work` or `/pause-work`
- **`--migrate --update` overwrites CLAUDE.md** — does not compare content when the file is already in template format
- **`--migrate --update` shows "not initialized" message** — the post-migration message prints unconditionally, even on update runs where the workspace is already initialized
