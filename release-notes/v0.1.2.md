# v0.1.2 Release Notes

**Date:** 2026-04-03

## Maintenance

### Rename /migrate skill to /workspace-update

Renamed the `/migrate` skill to `/workspace-update` to clarify its role. The `/migrate` skill conflated two concerns: initial migration (which is the CLI's job via `npx create-claude-workspace --migrate`) and ongoing template updates. The new `/workspace-update` skill focuses solely on applying template updates, with maintenance audits run before and after to catch integrity issues.

Fixed the CLI's post-migration message to be conditional: fresh migrations show the "run /workspace-init" prompt, update runs show "run /workspace-update to verify", and already-initialized workspaces skip the misleading "not initialized" message. Fixed a stale `/migrate` reference in the maintenance skill's example output.
