# v0.13.0-beta.0 Release Notes

**Date:** 2026-04-17

## What this is

First public beta of `@ulysses-ai/create-workspace` on npm. Shipped under the `beta` dist-tag — install with `npm install @ulysses-ai/create-workspace@beta` or scaffold with `npm create @ulysses-ai/workspace@beta`. The `latest` dist-tag remains unset until a stable `0.13.0` ships.

The work consolidated here was developed as a series of branches throughout 2026-04-17 (originally bumping through versions 0.12.0 → 0.12.3) but never pushed to npm. The version on the first publish jumps to `0.13.0-beta.0` to give the public beta a clean semver line aimed at a future stable `0.13.0`.

## Package identity

- **Scoped npm name:** `@ulysses-ai/create-workspace`. The bare `@ulysses` scope on npm is owned by a different user, so the maintainer claimed `@ulysses-ai` for the family of Ulysses-branded packages. The scope prefix carries the brand on npm, so the unscoped redundant `ulysses-` is dropped from the package name itself.
- **Bin name:** `create-workspace` (matches the `@vitejs/create-vite` → `create-vite` convention).
- **CLI banner:** `Ulysses Workspace` (friendly brand voice, replacing the package slug).
- **GitHub repo** stays at `ukt-solutions/create-ulysses-workspace` — the repo name carries product specificity so future workspace scaffolders under `ukt-solutions` can exist as first-class siblings. The asymmetry between npm scope and GitHub repo name is intentional architecture, documented in detail in the originally-shipped `branch-release-notes-revert-rename.md`.

Negative-path tests confirm all categories trigger correctly.

## CI publish workflow

- **`.github/workflows/publish.yml`** — tag-triggered publish via OIDC trusted publishing (no `NPM_TOKEN` secret to manage). Single workflow with tag-based dist-tag routing: `v*` matching `-beta.` publishes to `beta`, `-canary.` to `canary`, `-next.` to `next`, otherwise `latest`. Beta testers use `npm install @ulysses-ai/create-workspace@beta`; everyone else gets `latest`.
- **GitHub Releases** auto-created from the matching `release-notes/v{version}.md` file (or `--generate-notes` fallback). Pre-releases are correctly flagged in the GitHub UI.

## Earlier work also shipping in this beta

Two pieces from earlier in the workspace's development that landed on `main` after `v0.11.0` and never had their own published release docs:

- **Project-scoped Playwright MCP in scaffolded workspaces** (originally tagged for v0.9.x) — workspaces now ship with a project-scoped `.mcp.json` registering Playwright MCP with `--output-dir workspace-scratchpad/playwright`. The global Claude Code Playwright plugin is disabled at project scope so its writes don't leak across machines.
- **In-worktree session content** (originally tagged for v0.10.x) — the session tracker, specs, and plans now live inside each session's workspace worktree on the session branch (rather than at the launcher), so durable session thinking travels with the branch via `git push`. `/complete-work` removes them from the branch before the final PR. `migrate-session-layout.mjs` handles the one-time migration via `/workspace-update`.

Full prior release history is in `release-notes/v0.1.1.md` through `release-notes/v0.11.0.md`.

## Manual setup required before first publish via CI

The CI workflow uses npm trusted publishing, which requires a one-time configuration on npmjs.com:

1. Visit `https://www.npmjs.com/package/@ulysses-ai/create-workspace` → settings → Publishing
2. (For first publish, the package doesn't exist yet — alternative: configure via `npm` CLI, or do the first publish manually then add trusted publisher for future publishes)
3. Add a trusted publisher: `ukt-solutions/create-ulysses-workspace`, workflow `publish.yml`, environment (optional)

After that, `git push origin v0.13.0-beta.0` triggers the workflow.
