# v0.2.1 Release Notes

**Date:** 2026-04-03

## Fixes

### Workspace-update stall on pre-update audit

The `/workspace-update` skill's pre-update maintenance audit was a blocking gate that could stall the entire update flow. If Claude presented audit findings and the user didn't explicitly say "proceed," the update never happened — creating a loop where the bootstrap hook fired each session but the update never completed.

The audit is now informational only — it reports findings but always continues to the comparison and apply steps. The bootstrap hook also detects stale payloads (older than 5 minutes, meaning they survived at least one previous session) and escalates urgency with instructions to skip the audit entirely and proceed directly to applying changes.

Also fixed the `initialized` check in the bootstrap hook to accept `templateVersion` as a fallback signal for pre-flag workspaces.
