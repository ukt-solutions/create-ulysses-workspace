# v0.3.0 Release Notes

**Date:** 2026-04-05

## Features

### Persistent Work Sessions

Work sessions are now persistent, named entities that span multiple Claude Code chats. A work session has identity, state, and cross-chat continuity — you can exit a chat, start a new one, and resume where you left off. Multiple sessions can coexist in parallel across separate terminal windows.

**Workspace worktree model.** The workspace root stays on main and serves as the launcher. All real work happens in workspace worktrees. `/start-work` creates two worktrees per session — one for the workspace repo (`repos/{session}___wt-workspace/`) and one for the project repo (`repos/{session}___wt-{repo}/`). The root is effectively read-only: only `local-only-*` files and `.claude-scratchpad/` are writable from main. This replaces the old single-branch model where the workspace repo was checked out to a feature branch.

**Session markers.** Session state is tracked in `.claude-scratchpad/.work-session-{name}.json` with the session name, description, branch, repo, status, and contributing chat session IDs. The SessionStart hook surfaces active sessions when opening Claude at the workspace root. The SessionEnd hook marks chats as ended and writes safety-net entries to the inflight tracker.

**Inflight tracker.** Each session gets a living document at `shared-context/{user}/inflight/session-{name}.md` that accumulates context throughout the session's life. Skills update it at natural checkpoints. The SessionEnd hook appends a safety entry on clean exits. On resume, `/start-work` checks for uncaptured work by scanning message history files and filling gaps in the tracker.

**Helper scripts.** Two Node.js scripts (`create-work-session.mjs`, `cleanup-work-session.mjs`) consolidate mechanical git sequences into single invocations, reducing token usage by ~12 bash calls across the most common skill flows.

**Unified PR workflow.** `/complete-work` now presents both PRs (project + workspace) as a unified summary with a single "Merge both?" approval. One confirmation merges both, pulls both repos, and runs full cleanup.

**Session-aware hooks.** The PreCompact hook checks the inflight tracker's staleness before nudging — recently updated trackers get a lighter message. The repo-write-detection hook enforces workspace root restrictions, warning on writes outside `local-only-*` and `.claude-scratchpad/` from main.

**Skill rewrites.** `/start-work` supports resume, blank, and retroactive flows with workspace worktrees. `/pause-work` updates session markers and writes to the inflight tracker. `/handoff` and `/braindump` default to updating the inflight tracker when called within a session.

**Rule updates.** Worktree naming convention changed to `{session-name}___wt-{type}`. Workspace structure documents workspace worktrees and root write restrictions. Memory guidance distinguishes session-scoped context from cross-session knowledge.

## Known Issues

- The SessionStart hook does not register chat session IDs in the marker — registration is deferred to `/start-work`. If a user opens a workspace worktree without running `/start-work`, the chatSessions array may be inaccurate.

- Orphaned session marker cleanup is not explicitly added to the `/maintenance` skill. Existing stale worktree and inflight checks cover the same condition indirectly.

- Multi-repo work sessions (frontend + backend in one session) are not supported. Each session targets a single project repo. Multi-repo support requires a schema change (`repo` → `repos: []`). Planned for a future version.

- History reconstruction depends on message history files in `~/.claude/projects/`. The path format and session ID matching haven't been validated against all Claude Code versions. The feature should degrade gracefully if files aren't found.
