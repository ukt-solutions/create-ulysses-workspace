# v0.4.0 Release Notes

**Date:** 2026-04-06

## Features

### Multi-repo work sessions

Work sessions now bind to multiple project repos. A single session can span any combination of repos defined in workspace.json â€” frontend + backend, app + platform, or whatever the task requires.

The session marker schema changes from `repo` (string) to `repos` (array). The inflight tracker follows the same pattern. No backwards compatibility shim â€” the project has no external users.

**Scripts:** `create-work-session.mjs` accepts comma-separated repos and creates one worktree per repo. `cleanup-work-session.mjs` loops over all project worktrees and branches. A new `add-repo-to-session.mjs` script adds a repo to an existing session mid-flight â€” creating the branch, worktree, and updating markers.

**Skills:** Four skills updated to iterate over `marker.repos`: start-work (multi-repo selection and mid-session addition), complete-work (per-repo rebase, release notes, push, and atomic merge), pause-work (per-repo push and draft PR), sync-work (push loop).

**Hook:** `repo-write-detection` extended to detect writes to repos in workspace.json but not in the active session, nudging Claude to offer adding the repo.

### `/aside` skill â€” drive-by idea capture

New skill for capturing stray ideas without interrupting the current conversation.

- **Default (researched):** Dispatches the `aside-researcher` subagent in the background. The agent searches workspace, project repos, and web, then writes a single `local-only-{slug}.md` file with the user's original thought preserved verbatim alongside its findings.
- **Quick (`--quick`):** No subagent. Writes the thought directly to a local-only file with a Further Investigation section.

Files are always `local-only-*` (gitignored), session-agnostic, and discoverable by `/promote` and `/maintenance`. The `variant: aside` frontmatter distinguishes them from other local-only files.

**New files:**
- `template/.claude/agents/aside-researcher.md` â€” Sonnet agent with Read/Glob/Grep/WebSearch/WebFetch/Write tools.
- `template/.claude/skills/aside/SKILL.md` â€” Skill definition with quick and research mode flows.

**Deprecation:** `/braindump side` now redirects to `/aside`.

## Known Issues

- **`add-repo-to-session` untested with real multi-repo workspace.** The script was written and reviewed but integration testing only verified single-repo create/cleanup. Full multi-repo testing requires a workspace with multiple repos in workspace.json.
- **Repo-write-detection hook path matching.** Edge cases with repo names containing special characters or nested paths haven't been tested.
- **`/aside` skill discovery in worktrees.** The skill didn't appear in the skills list during a worktree test session. Resolved after merge to main â€” root cause unknown. May be a caching or timing issue.
- **`/aside` smoke test not completed.** Quick mode, research mode, and braindump redirect were not tested end-to-end.
