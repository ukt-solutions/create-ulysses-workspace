# v0.4.1 Release Notes

**Date:** 2026-04-06

## Fixes

`/complete-work` now cleans up the inflight tracker after synthesizing release notes. Previously, the tracker file at `shared-context/{user}/inflight/session-{name}.md` was left behind when a session completed — worktrees, branches, and session markers were removed, but the inflight file accumulated across sessions. Step 6 of the skill now removes it as part of source consumption.

`/start-work` now captures prior conversation context into the inflight tracker. When brainstorming, spec writing, or design discussion happens before `/start-work` is called, that reasoning is summarized and written into the tracker after session creation. This fixes a pipeline gap where `/complete-work` produced thin release notes because the design decisions from the brainstorming phase were never captured into the session's data.

## Maintenance

Added a complete documentation system: 11 textbook chapters organized in four parts (Concepts, The Toolkit, Lifecycle, Practice) and 3 audience-specific guides (Solo Developer, Team Lead, New Team Member). Documentation lives at `docs/` in the repo root — it is reference material, not scaffolded into workspaces.

The chapters follow an authoritative, concept-first voice influenced by Spring's structure, Stripe's directness, and Next.js's clarity. The guides use a warm, hand-holding tone accessible to junior developers. Both tiers are designed for NotebookLM audio generation.

Added a README with quick start instructions and a documentation table linking to all chapters and guides.

## Known Issues

- Chapter word counts average ~1,200 words against a 1,500-2,000 spec target. Content is substantively complete — code blocks and diagrams add visual weight not captured in word count — but a review pass could expand thin sections.
- Docusaurus wrapper not yet configured. The docs are markdown files ready for a static site generator. A follow-up task to add Docusaurus config would produce an HTML documentation site.
- Documentation uses the working name `create-claude-workspace`. A search-and-replace pass will be needed when the product naming decision (#38) is made.
