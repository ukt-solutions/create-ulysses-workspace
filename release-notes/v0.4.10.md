# v0.4.10 Release Notes

**Date:** 2026-04-07

## Fixes

Added a mandatory override section to workspace-structure rule that explicitly enforces spec/plan locations in the project worktree root (`design-{topic}.md`, `plan-{topic}.md`). External skills like Superpowers brainstorming default to `docs/superpowers/specs/` — the new rule is worded as a hard override that takes priority. Prevents specs from being written to incorrect locations that the release notes pipeline can't find.
