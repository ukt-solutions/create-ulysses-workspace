# v0.4.11 Release Notes

**Date:** 2026-04-07

## Fixes

Added "No Retry Loops" section to the honest-pushback rule. When a fix attempt fails twice with the same or similar result, Claude must stop, state what it expected vs what happened, identify the gap in understanding, research the issue, and present findings before trying again. Prevents trial-and-error token burn when reading docs would resolve the issue in one turn.

Also added mandatory override section to workspace-structure rule enforcing spec/plan locations in the project worktree root (`design-{topic}.md`, `plan-{topic}.md`), explicitly overriding external skill defaults like Superpowers' `docs/superpowers/specs/` path.
