# v0.4.12 Release Notes

**Date:** 2026-04-07

## Features

Added "Verify, Don't Assume" section to the honest-pushback rule. When evidence is available (logs, database, UI, API responses, runtime state), Claude must check rather than guess. Includes guidance on adding temporary instrumentation when logs aren't verbose enough. Claude asks the user once whether to verify proactively, then remembers the preference for the session — balancing thoroughness with productivity.
