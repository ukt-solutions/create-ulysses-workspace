# v0.4.13 Release Notes

**Date:** 2026-04-07

## Fixes

Tightened chatSessions tracking in the start-work skill. Each chat session entry now uses the actual UUID from the Claude Code conversation log file (`~/.claude/projects/{path}/{uuid}.jsonl`) as the `id` field, with clear instructions on how to discover it. The `names` field is now an array to track session renames over time. Previously, the skill said "session ID" without specifying where to get it, leading to session names being used as identifiers — which break when renamed.

The session-end hook already used the correct UUID from `input.session_id`. History reconstruction now explicitly references the `id` field for log file lookups.
