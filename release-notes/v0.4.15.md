# v0.4.15 Release Notes

**Date:** 2026-04-07

## Fixes

Replaced the `.current-chat-id` scratchpad file approach with direct session marker registration in the session-start hook. The hook now reads `input.session_id` (UUID from Claude Code) and writes it directly into the active work session's `chatSessions` array in the marker. This is correct for parallel sessions — each worktree's hook writes to its own session marker.

The start-work skill no longer needs to handle chat registration for resumed sessions (the hook does it). For new sessions (blank flow), the skill still registers the first chat manually since the session didn't exist when the hook fired.

Removed `getCurrentChatId()` from _utils.mjs and the `.current-chat-id` scratchpad file — both superseded by the hook-to-marker approach.
