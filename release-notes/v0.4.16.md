# v0.4.16 Release Notes

**Date:** 2026-04-09

## Features

`/complete-work` now checks for `package.json` versioning in each project repo before merging. If a repo has a `version` field, the skill determines the appropriate bump from the release note type: fixes/chores auto-bump patch, features suggest minor (with user confirmation), breaking changes suggest major (with user confirmation). Repos without `package.json` or without a version field are skipped — versioning is opt-in.
