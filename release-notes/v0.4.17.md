# v0.4.17 Release Notes

**Date:** 2026-04-09

## Fixes

### Windows compatibility (6 fixes)

1. **Hook commands: `$CLAUDE_PROJECT_DIR` backslash workaround** (settings.json) — All hook commands now use `cygpath -u` on Windows (Git Bash) to convert backslash paths to Unix-style before passing to `node`. On Unix, the path passes through unchanged. Works around [claude-code#21878](https://github.com/anthropics/claude-code/issues/21878).

2. **Symlinks: junction fallback on Windows** (create-work-session.mjs) — The `repos/` symlink in workspace worktrees now uses `'junction'` type on Windows (works without elevation) with an absolute path target. Unix continues using relative symlinks.

3. **Path normalization in repo-write-detection** (repo-write-detection.mjs) — All paths extracted from tool input are normalized to forward slashes before matching. Prevents the write-detection hook from silently passing through writes on Windows where paths use backslashes.

4. **Quoted paths in git clone** (lib/git.mjs) — Both remote URL and target directory are now quoted in the `git clone` command. Fixes paths with spaces (common on Windows: `C:\Users\First Last\...`).

5. **`path.basename()` for workspace name** (lib/init.mjs) — Replaced `targetDir.split('/').pop()` with `path.basename(targetDir)`. The split approach returned the full Windows path instead of just the directory name.

6. **Shebang lines** — No change needed. Claude Code calls hooks via `node "path"` (explicit in settings.json), so shebangs are informational only.
