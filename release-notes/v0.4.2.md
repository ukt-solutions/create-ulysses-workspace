# v0.4.2 Release Notes

**Date:** 2026-04-06

## Fixes

Fixed package.json version not being updated during releases. The CLI reads `package.json` to determine the template version for upgrade staging, but the version was stuck at 0.2.1 while the actual template had progressed to 0.4.1. This caused the `--upgrade` command to display incorrect version numbers and "already up to date" messages.

The `/release` skill now includes a Step 7b that bumps `package.json` as part of the release flow, preventing this drift from recurring.
