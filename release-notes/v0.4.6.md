# v0.4.6 Release Notes

**Date:** 2026-04-06

## Fixes

Rewrote `--init` to install the full `.claude/` directory directly instead of bootstrapping from the payload. The previous approach (stage payload, wire a hook, hope Claude reads the skill from an @-reference) failed in practice — the hook didn't fire reliably, `@` references to `.workspace-update/` paths didn't resolve, and Claude ignored the bootstrap CLAUDE.md instructions.

The new init:
- Copies `.claude/` directly (skills, hooks, scripts, agents, rules) — all 13 skills available immediately
- Creates `shared-context/`, `repos/`, `.claude-scratchpad/`
- Generates `workspace.json` with full default settings and the template version
- Generates `CLAUDE.md` from the template (with project name substituted)
- Backs up existing CLAUDE.md to `.bak`
- Merges template `.gitignore` entries
- Still stages the payload (for `/workspace-update` to verify integrity later)

Next step after init is now simply `/setup` — no intermediate workspace-init dance.
