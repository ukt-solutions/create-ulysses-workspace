# v0.4.9 Release Notes

**Date:** 2026-04-06

## Features

### workspace-init: Chat history scanning (Step 8)
Scans `~/.claude/projects/` for prior Claude Code conversation logs related to this project. Builds a processing manifest (`.claude-scratchpad/chat-history-manifest.json`) before reading content to survive auto-compaction. Extracts decisions, architectural choices, work-in-progress context, and convention candidates. Synthesizes into shared context with user approval.

### workspace-init: Worktree formalization (Step 11)
Detects existing git worktrees from pre-workspace development and formalizes them as work sessions. Creates session markers, inflight trackers, and workspace worktrees. Searches chat history for sessions that touched each branch to populate the tracker with prior context.

### workspace-init: Team member onboarding (Step 16)
The "Set up workspace remote" step now handles three cases:
- **Remote exists:** skip
- **Connecting to existing repo:** `git remote add`, `git fetch`, `git rebase` with conflict resolution guidance for merging your extractions with a teammate's committed workspace
- **Creating new:** `gh repo create` as before

## Fixes

### workspace-init: Skip already-extracted content (Step 7)
Before fetching from any documentation source, checks `shared-context/` and `.claude/rules/` for files that already contain extracted content from that source. Asks before re-fetching. Prevents duplicate extraction when content was pulled down in a previous session or by a teammate.
